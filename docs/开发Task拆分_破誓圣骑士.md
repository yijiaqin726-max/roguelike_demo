# 开发 Task 拆分：破誓圣骑士 Demo

## 1. 拆分目标

这份 Task 拆分不是按“先把肉鸽机制堆完整”来排，而是按“先做出一个能被当作完整 Demo 打开的产品”来排。

当前版本的优先级原则如下：

1. 先补齐产品外壳：首页、开始游戏、继续游戏、层数显示、保存与读档
2. 再补齐核心创新：守誓 / 破誓、腐化值、黑暗脉冲
3. 再补齐展示层：HUD、升级界面、战斗反馈、美术统一
4. 不影响 Demo 成立的长期扩展内容统一后置，并明确标注为 `可添加`

这份文档的目的，不只是列功能，而是帮助我们判断：

- 什么必须先做，才能让项目像一个完整游戏
- 什么必须尽快做，才能让创新点被面试官看见
- 什么内容现在不做也不会影响 Demo 成立

## 2. 当前项目已有基础

目前项目中已经存在的核心内容包括：

- 玩家移动
- 自动攻击
- 敌人追击与刷怪
- 子弹伤害
- 经验与升级
- 升级弹窗
- 玩家血量与 HUD
- 基础场景与部分素材替换

这意味着我们接下来不是从零开始开发，而是在已有原型上做两件事：

1. 把它补成一个“可进入、可继续、可展示”的完整 Demo
2. 把“破誓圣骑士”的创新点真正落地成系统

## 3. 排序原则

### 3.1 优先级判断标准

一个任务是否应该提前做，优先看这四件事：

1. 它是否影响“这个项目看起来像不像一个完整游戏”
2. 它是否影响“面试官能不能快速理解创新点”
3. 它是否影响“玩家下次进入时能不能继续爬塔”
4. 它是否影响“截图、演示、汇报时的第一印象”

### 3.2 当前默认策略

- 首页与保存系统优先级高于继续扩展战斗机制
- 创新点优先级高于普通数值强化
- 美术素材与 UI 问题要显式写入 Task，而不是最后再补
- 对 Demo 不是刚需的扩展内容统一排到最后，并标记为 `可添加`

## 4. 第一阶段：先做出完整产品外壳

这一阶段的目标是：让项目打开后像一个真正可进入、可继续、可爬塔的游戏，而不是直接跳进测试场景。

### A1. 首页 Scene

目标：
项目启动后先进入首页，而不是直接进入战斗 Scene。

任务内容：

- 新建首页 Scene
- 首页至少包含：
  - 游戏标题
  - 开始游戏
  - 继续游戏
  - 退出游戏
- 首页文案与视觉要体现“破誓圣骑士”主题，而不是通用肉鸽标题页

验收标准：

- 启动游戏后先进入首页
- 可以从首页进入正式战斗场景

### A2. 开始游戏流程

目标：
点击“开始游戏”后，总是从新开局开始。

任务内容：

- 点击开始游戏时：
  - 新建存档数据
  - 重置当前局内进度
  - 设置当前层数为第 1 层
  - 进入战斗场景

验收标准：

- 每次点击开始游戏都从第 1 层开始

### A3. 继续游戏流程

目标：
点击“继续游戏”后，从上次保存的爬塔进度继续。

任务内容：

- 首页读取本地存档
- 如果没有存档，则继续游戏按钮置灰或不可点击
- 如果存在存档，则进入上次保存的层数和进度

验收标准：

- 有存档时可以继续
- 无存档时不会报错，且按钮状态合理

### A4. 保存与读档系统

目标：
玩家关闭游戏后，下次仍能从当前爬塔层数继续。

任务内容：

- 设计最小存档结构，至少保存：
  - 当前层数
  - 玩家基础成长状态
  - 当前腐化值
  - 是否解锁继续游戏
- 建议保存时机：
  - 进入下一层时自动保存
  - 返回首页时保存

验收标准：

- 关闭游戏后重新运行，继续游戏仍能读取正确层数

### A5. 当前层数显示

目标：
玩家在战斗中始终知道自己打到第几层。

任务内容：

- 在战斗 HUD 中新增层数显示
- 文案建议：
  - `第 1 层`
  - `第 2 层`
  - 或更风格化的 `深渊回廊 1F`

验收标准：

- 游戏中能清楚看到当前层数
- 进入下一层后层数立即更新

### A6. 关卡完成与进入下一层

目标：
形成基础“爬塔”体验，而不是永远停留在同一层测试场。

任务内容：

- 明确一层结束条件
- 结束后进入下一层
- 下一层可以使用新的随机种子或新的刷怪参数

验收标准：

- 玩家能明确感受到自己在推进层数
- 爬塔进度和存档系统能正确联动

## 5. 第二阶段：落地核心创新点

这一阶段的目标是：让“破誓圣骑士”不只是题材，而是真正改了成长规则的系统。

### B1. 腐化值系统

目标：
建立破誓系统的核心资源骨架。

任务内容：

- 新增玩家腐化值系统
- 至少包含：
  - 当前腐化值
  - 最大腐化值
  - 腐化阶段划分
  - 增加与降低接口

验收标准：

- 运行时能正确增加 / 降低腐化值

### B2. 升级系统改造成守誓 / 破誓

目标：
把当前普通升级弹窗改造成创新点入口。

任务内容：

- 替换现有普通升级选项文案与逻辑
- 升级界面至少保留：
  - 守誓
  - 破誓
  - 第三个按钮作为预留位或暂未开放位

验收标准：

- 玩家第一次升级时，就能看出这不是普通数值三选一

### B3. 守誓效果

目标：
建立稳定、安全、可控的成长路线。

任务内容：

- 守誓效果建议包括：
  - 回复生命
  - 提供护盾或减伤
  - 降低腐化或压制失控概率

验收标准：

- 玩家能明确理解守誓代表“稳路线”

### B4. 破誓效果

目标：
建立高收益、高风险的成长路线。

任务内容：

- 破誓效果建议包括：
  - 增加腐化值
  - 强化攻击表现
  - 提升黑暗能力的触发强度

验收标准：

- 玩家第一次破誓后能立刻感觉到更强

### B5. 黑暗脉冲

目标：
把“腐化越高越危险”做成可见机制，而不是纯文案说明。

任务内容：

- 攻击时，根据腐化值概率触发额外黑暗脉冲
- 腐化越高，触发概率或效果越强

验收标准：

- 中高腐化阶段能明显看到随机黑暗脉冲

## 6. 第三阶段：把创新点做成能展示的 Demo

这一阶段的目标是：不只是功能成立，还要让面试官在截图和演示里一眼看懂“创新点是什么”。

### C1. 腐化 HUD

目标：
让玩家和面试官都能一眼看到创新系统。

任务内容：

- 在 HUD 中新增腐化值显示
- 形式可选：
  - 腐化条
  - 分段腐化条
  - 数值 + 条形

验收标准：

- 玩家在战斗中能随时读懂当前腐化状态

### C2. 升级界面重写

目标：
让升级弹窗服务创新点表达，而不是只是按钮列表。

任务内容：

- 守誓按钮文案突出“稳定、净化、控制”
- 破誓按钮文案突出“堕落、强化、风险”
- 第三个按钮如果未启用，要明确写成“未开放”或“可添加”

验收标准：

- 不看代码也能看懂升级系统设计意图

### C3. 黑暗脉冲表现优化

目标：
让失控风险可以被截图和演示出来。

任务内容：

- 为黑暗脉冲加入更明显的视觉反馈，例如：
  - 黑色冲击圈
  - 暗色爆发
  - 命中波纹

验收标准：

- 黑暗脉冲可以通过截图明确展示

## 7. 第四阶段：美术素材与 UI 任务

这一阶段明确把“美工素材问题”纳入排期，而不是默认最后补。

### D1. 首页美术与 UI 素材

目标：
首页看起来像一个有主题的产品入口，而不是 Unity 默认按钮页面。

任务内容：

- 首页背景
- 标题排版
- 按钮风格统一
- 开始 / 继续 / 退出按钮状态区分

要求：

- 风格与战斗界面一致
- 不使用默认 Unity 白按钮原样式

### D2. 战斗内素材统一

目标：
让玩家、敌人、经验球、子弹、地板在视觉上属于同一产品。

任务内容：

- 统一玩家、敌人、地板、经验球、子弹的视觉层级和风格
- 明确当前素材缺口，并补齐最影响观感的部分

当前已知素材问题：

- 玩家与敌人已接入 Kenney 风格
- 子弹仍偏占位感，后续需要进一步优化
- 经验球保留为绿色小球，可继续微调表现

### D3. 破誓主题表现

目标：
让角色从圣骑士向破誓者转变时，在视觉上也有成长轨迹。

任务内容：

- 腐化值升高后，角色或 HUD 出现暗化、黑化或污染趋势
- 在视觉上体现“从守誓者到破誓者”的变化

### D4. HUD 与升级界面统一风格

目标：
血条、层数、腐化条、升级弹窗属于同一套界面语言。

任务内容：

- 统一血条、层数、腐化值、升级弹窗面板的配色与材质风格
- 避免默认按钮、默认文本、默认面板混搭

## 8. 第五阶段：数值、演示与面试支持

### E1. 第一轮数值调优

目标：
在 1 到 2 分钟内让玩家体验到创新点。

任务内容：

- 调整第一次升级出现时间
- 调整敌人刷新节奏
- 调整腐化增长速度
- 调整黑暗脉冲触发频率

验收标准：

- 玩家能在较短时间内体验到“守誓 / 破誓 / 腐化 / 失控”

### E2. 截图与演示节点

目标：
让项目能快速产出用于面试汇报的演示素材。

任务内容：

- 首页截图
- 层数显示截图
- 守誓 / 破誓升级截图
- 高腐化黑暗脉冲截图
- 继续游戏截图

### E3. 面试演示路径

目标：
让项目不只是能玩，还能被清楚讲述。

任务内容：

- 从首页进入游戏
- 展示层数
- 展示继续游戏
- 展示第一次破誓
- 展示中高腐化战斗

## 9. 推荐实际开发顺序

为了保证 Demo 尽快成立，推荐按下面顺序推进：

1. 首页 Scene
2. 开始游戏流程
3. 保存与继续游戏
4. 当前层数显示
5. 关卡完成与进入下一层
6. 腐化值系统
7. 升级系统改造成守誓 / 破誓
8. 守誓效果
9. 破誓效果
10. 黑暗脉冲
11. 腐化 HUD
12. 升级界面重写
13. 首页与战斗界面美术统一
14. 第一轮数值调优
15. 截图与面试演示整理

## 10. 下一步最值得立刻开工的 Task

如果只看“现在立刻做什么最划算”，建议从下面开始：

1. 做首页 Scene
2. 接开始游戏 / 继续游戏逻辑
3. 设计并接入最小存档结构
4. 在战斗 HUD 中显示当前层数
5. 实现一层结束并进入下一层

做完这一轮后，再进入创新机制主线。

## 12. 当前默认判断

为了保证 Demo 尽快成立，当前默认采用以下判断：

- 先做首页和保存，不先扩更多战斗细节
- 先做“爬塔可继续”的产品闭环，再做复杂 Roguelike 扩展
- 先确保“破誓圣骑士”这一条创新路线成立，不急着扩多职业
- 美术素材和 UI 问题必须写入 Task，而不是最后补
- 所有不影响 Demo 成立的内容统一后置，并明确标记为 `可添加`
