## 1. 今日目标变化

今天项目方向从“先补一个可截图的割草 Demo”进一步收束为：

- 目标不是复刻一个普通肉鸽
- 目标是做一个能支撑游戏策划面试的 Demo

- 重点是“这个项目相对市面肉鸽的创新点是什么，以及如何被展示出来”

后续所有开发、文档和演示，都要围绕这个目标服务。

## 2. 已确定的核心项目定位

### 2.1 项目题材方向

项目当前创新主题已经确定为：

**破誓圣骑士（Oathbreaker Paladin）**

不是普通圣骑士成长，而是围绕“破誓、腐化、失控风险”展开。

### 2.2 项目一句话创新描述

当前可以稳定使用的核心表达为：

**这不是一个普通肉鸽，而是一个围绕“破誓腐化”展开构筑的动作 roguelike。**

玩家通过升级时的选择主动背弃誓言，积累腐化值，换取更强的黑暗力量，但也会承担越来越高的失控风险。

### 2.3 创新点与普通肉鸽的区别

已明确的设计结论：

- 创新点不能只是 DND / 博德之门换皮
- 创新点必须落在“规则改动”上，而不是题材包装上
- 当前项目真正改动的是“升级系统的意义”

传统肉鸽常见逻辑：

- 升级 = 选更强的数值或技能

当前项目逻辑：

- 升级 = 在“守誓 / 破誓”之间做身份抉择
- 破誓后获得更强力量，但战斗控制权下降

这被认为是当前项目最值得面试展示的创新核心。

## 3. 已确定的创新机制结论

### 3.1 守誓 / 破誓

升级时不再是普通三选一，而是至少包含：

- 守誓
- 破誓
- 第三个按钮可作为预留位或未开放位

### 3.2 腐化值

破誓会持续增加腐化值。

腐化值越高：

- 玩家越强
- 黑暗能力越明显
- 失控风险越高

### 3.3 失控风险类型

失控风险已经确定不是“单纯掉血”或“纯构筑锁死”，而是：

**随机释放黑暗脉冲**

也就是：

- 玩家攻击时，有概率额外爆发黑暗脉冲
- 腐化越高，触发越频繁或效果越强
- 这体现“力量增强，但控制权下降”

### 3.4 守誓路线意义

守誓不是废选项，当前设计结论是：

- 守誓 = 稳定收益 + 控制腐化

也就是：

- 回血
- 护盾 / 减伤
- 降低腐化或压制失控风险

这样才能和破誓路线形成真正选择，而不是伪选择。

## 7. 今天达成的重要策划共识

### 7.1 关于肉鸽基本功

已明确的面试表达结论：

- 肉鸽的核心不是“随机越多越好”
- 随机的作用是让玩家每局重新做决策
- 肉鸽策划必须理解：
  - 核心循环
  - 局内成长
  - 随机与可控
  - 数值节奏
  - 风险收益
  - 重玩价值

### 7.2 关于“创新点”怎么回答面试官

今天明确形成的表达方式：

- 行业共性不是创新点
- 创新点应该回答“我改了哪条规则”
- 如果去掉题材包装后还能成立，才更像真创新

当前项目的标准回答模板已经形成：

“我的创新点不是 DND 题材本身，而是把传统肉鸽偏数值化的升级选择，改造成了守誓 / 破誓的身份抉择。玩家每次升级不是只拿更强的 buff，而是在力量增长和控制权丧失之间做风险收益判断。”

## 8. 当前开发优先级结论

今天最后确认的优先级，不是先做腐化机制，而是：

### 第一优先级：产品闭环

先做这些：

1. 首页 Scene
2. 开始游戏
3. 继续游戏
4. 最小存档结构
5. 当前层数显示
6. 关卡完成并进入下一层

原因：

- 先让项目像一个完整游戏
- 先形成“爬塔可继续”的产品壳
- 这样后面的创新点才有承载空间

### 第二优先级：创新机制

在产品外壳完成后，继续做：

1. 腐化值系统接入
2. 守誓 / 破誓升级界面
3. 守誓效果
4. 破誓效果
5. 黑暗脉冲

### 第三优先级：展示层

之后再做：

1. 腐化 HUD
2. 升级界面文案与风格
3. 黑暗脉冲视觉反馈
4. 美术与 UI 统一
